# Ore Slice Classifier — «Скажи мне, кто твой шлиф»

Интерпретируемая ML/CV-система для анализа OM-изображений полированных шлифов руды. Решение классифицирует руду, обрабатывает панорамные снимки по тайлам, визуализирует области решений модели и формирует лабораторные отчёты.

> Проект разработан в рамках хакатона по задаче автоматизации анализа технологических свойств рудного сырья.

## Задача

Ручной анализ полированных шлифов зависит от опыта геолога, занимает много времени и плохо масштабируется на большие партии изображений. Цель проекта — автоматизировать первичную классификацию руды и предоставить специалисту интерпретируемый результат, который можно визуально проверить.

Система работает с тремя классами:

| ML-класс | Результат | Геологический смысл |
|---|---|---|
| `ordinary` | Рядовая руда | Преобладают обычные срастания |
| `difficult` | Труднообогатимая руда | Преобладают тонкие срастания |
| `talc` | Оталькованная руда | Доля тайлов с признаками талька выше заданного порога |

Экспертная логика итоговой классификации:

1. Если оценочная доля талька больше 10% — **оталькованная руда**.
2. Если талька не больше 10% и преобладают обычные срастания — **рядовая руда**.
3. Если талька не больше 10% и преобладают тонкие срастания — **труднообогатимая руда**.

Порог талька можно изменять в интерфейсе.

## Возможности решения

- загрузка JPEG, PNG, TIFF и нескольких файлов одновременно;
- выбор одной из трёх обученных PyTorch-моделей;
- тайловая обработка панорам с настраиваемыми размером и перекрытием;
- итоговая классификация по экспертной логике;
- цветовая карта классов: зелёный — обычные, красный — тонкие, синий — тальк;
- карта уверенности модели;
- оценка долей классов и светлых сульфидных областей;
- таблица вероятностей для каждого тайла;
- ручная коррекция классов геологом для active learning;
- пакетная обработка серии изображений;
- экспорт CSV, PDF, PNG и GeoJSON;
- локальная работа с конфиденциальными данными;
- безопасная обработка больших изображений в условиях ограниченной памяти.

## Архитектура

```text
Изображение / панорама
          │
          ▼
Безопасное чтение и ограничение размера
          │
          ▼
Разбиение на перекрывающиеся тайлы

          ▼

          │

          │
          ▼

```

Основная логика inference находится в [`src/inference.py`](src/inference.py), пользовательский интерфейс — в [`app/streamlit_app.py`](app/streamlit_app.py).

### Этапы подготовки

Первый notebook выполняет следующий пайплайн:

1. Рекурсивно находит поддерживаемые изображения.
2. Собирает путь, формат, размер, разрешение, цветовой режим и читаемость.
3. Нормализует названия исходных папок в классы `ordinary`, `difficult`, `talc`.
4. Рассчитывает SHA-256 для поиска точных дубликатов.
5. Находит одинаковые изображения с конфликтующими метками.
6. Исключает конфликтные примеры из обучающего манифеста.
7. Сопоставляет исходные изображения с цветными аннотациями талька.
8. Формирует воспроизводимое разбиение train/validation/test.
9. Экспортирует каталоги и отчёт об аудите.

Разбиение датасета:

- train — 70%;
- validation — 15%;
- test — 15%.

Точные дубликаты группируются до разбиения, чтобы одинаковый файл не оказался одновременно в train и test. Seed разбиения — `42`.

## ML-модели

Обучение и сравнение моделей выполнено в [`notebooks/02_ore_models_training_project_structure.ipynb`](notebooks/02_ore_models_training_project_structure.ipynb).

### Предобработка

Все изображения приводятся к размеру `224×224` и нормализуются статистиками ImageNet. Для train используются аугментации:

- горизонтальное и вертикальное отражение;
- случайный поворот до 15°;
- изменение яркости, контраста, насыщенности и оттенка.

Для компенсации дисбаланса классов применяются `WeightedRandomSampler` и веса классов в `CrossEntropyLoss`.

### Обученные модели

| Модель | Назначение | Лучший validation accuracy | Лучший validation macro-F1 |
|---|---|---:|---:|
| TinyCNN | Лёгкий baseline | 0.618 | 0.580 |
| MobileNetV3 Small | Быстрый inference | 0.800 | 0.783 |
| ResNet18 | Основная модель | **0.885** | **0.872** |

Модели обучались 5 эпох с оптимизатором AdamW. Лучшие веса выбирались по validation macro-F1.

### Test-результаты ResNet18

На отложенной тестовой выборке из 165 изображений:

| Метрика | Значение |
|---|---:|
| Accuracy | 0.818 |
| Macro-F1 | 0.802 |
| F1 — рядовая руда | 0.797 |
| F1 — труднообогатимая руда | 0.853 |
| F1 — оталькованная руда | 0.757 |

Истории обучения и test-предсказания находятся в `reports/`. Веса моделей находятся в `models/`.

## Streamlit-приложение

### Тайловый inference

Панорама разбивается на перекрывающиеся тайлы. Тайлы обрабатываются пакетами, после чего для каждого сохраняются:

- координаты;
- предсказанный класс;
- вероятности всех трёх классов;
- уверенность модели;
- приближённая доля светлых слабонасыщенных пикселей.

Тайлы обрабатываются потоково: в памяти одновременно находится только один batch. Это уменьшает потребление RAM на Streamlit Cloud.

Изображения больше 30 мегапикселей автоматически уменьшаются до безопасного размера. Интерфейс сообщает пользователю о таком преобразовании.

### Интерпретируемость

Приложение показывает:

- исходное изображение;
- цветную tile-карту;
- карту уверенности;
- долю тайлов с признаками талька;
- доли обычных и тонких срастаний среди нетальковых тайлов;
- CV-оценку светлых сульфидных областей;
- среднюю уверенность;
- подробную таблицу тайлов.

Геолог может изменить класс отдельного тайла и выгрузить исправления в CSV для последующего дообучения.

### Экспорт

| Формат | Содержимое |
|---|---|
| CSV | Итоговые количественные метрики |
| PDF | Краткий лабораторный отчёт с картой и таблицей |
| PNG | Цветная карта классификации |
| GeoJSON | Полигоны тайлов и вероятности классов |
| Batch CSV | Сводка по загруженной партии |

GeoJSON использует пиксельные координаты. Для географической привязки необходимы метаданные масштаба и CRS.

```bash

cd Ore-slice-classifier
```

Рекомендуется Python 3.11–3.13.

Windows PowerShell:

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
```

```bash

source .venv/bin/activate
```


Для запуска интерфейса исходный датасет не требуется: веса моделей уже находятся в репозитории.

## Ограничения

Текущие нейронные сети являются **классификаторами изображений**, а не моделями пиксельной сегментации. Поэтому:

- цветная карта показывает классы тайлов, а не точные границы минералов;
- доля талька является оценкой по площади классифицированных тайлов;
- доли типов срастаний рассчитываются по нетальковым тайлам;
- доля сульфидов является CV-приближением по светлым слабонасыщенным пикселям;
- GeoJSON не содержит физического масштаба и CRS без метаданных прибора.

Эти ограничения обозначены в интерфейсе и PDF-отчёте. Результат предназначен для поддержки эксперта, а не для полной замены лабораторной верификации.

## Направления развития

- разметка точных пиксельных масок и обучение U-Net/SegFormer;
- оценка площади талька с требуемой ошибкой ±3%;
- group split по идентификатору физического шлифа;
- hard-negative mining и дообучение на исправлениях геолога;
- калибровка вероятностей и порог отказа на неуверенных примерах;
- полноценная поддержка масштаба, метаданных микроскопа и GIS CRS;
- фоновая очередь для пакетной обработки гигапиксельных TIFF;
- версионирование данных и моделей через DVC или MLflow.
